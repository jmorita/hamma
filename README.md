# 韓麻 (ハンマ) — 韓国式麻雀 WEBアプリ

韓国パラダイスシティで2026年2月にリリースされた韓国式麻雀『韓麻』のルールを実装した、
CPU3人と対戦できるブラウザ麻雀です。

ルール典拠: [麻雀界Web「ざっくり韓麻ルール解説」](https://mj-news.com/news/info-20260228/)

## 動かす

```bash
npm install
npm run dev        # http://localhost:5173
npm test           # コアロジックのテスト
npm run typecheck
```

画面は3つのタブに分かれています。

| タブ | 内容 |
|---|---|
| 卓 | 対局。麻雀卓を上から見た配置で、席ごとに牌の向きが変わる |
| ルール | 全ルールの一覧。点数まわりは実際の設定値を表示するので説明と実装がずれない |
| 設定 | 3人/4人、リーチ加点、仮想レート/レーキ/デポジット |

開発中は `http://localhost:5173/#tiles` で34種すべての牌の絵柄を確認できます
(本番ビルドには含まれません)。

## 実装したルール

| 項目 | 内容 |
|---|---|
| 人数 | 3人打ち / 4人打ち |
| 使用牌 | 136枚。3人打ちでも枚数は変わらない。数牌の5は全12枚が赤ドラ |
| 和了形 | 4メンツ1雀頭ならどんな形でも和了。役なし・符なし |
| 例外形 | 七対子・国士無双のみ特別に認める (門前限定) |
| 表ドラ | 開局時から2枚オープン |
| 裏ドラ | リーチ和了時に2枚オープン |
| カンドラ | 無し (カンしてもドラは増えない) |
| 副露 | ポン・カンのみ。**チーは無い** |
| カン制限 | 1局で全員合わせて4回まで。上限到達後はカンできないだけで流局しない |
| 点数 | ロン6 / ツモ2 / リーチ2 / ドラ1枚1点 / 国士無双20点 |
| 精算 | ロンは放銃者が全額。ツモは全員が同額 → 4人打ちなら収入は3倍、3人打ちなら2倍 |
| 支払い上限 | 1人あたり20点 |
| 複合 | 国士無双はドラ・リーチと複合しない。七対子はドラと複合する |
| フリテン | **概念なし**。テンパイしていればいつでも和了れる |
| ダブロン | 頭ハネ (放銃者から見て手番が近い席が総取り) |
| リーチ | 門前かつテンパイで宣言可。**供託棒なし** (宣言牌を横に置くだけ) |
| 親 | 第一ツモを取るだけ。連荘なし・席替えなし |
| 流局 | ツモ山が尽きたら流局。ノーテン罰符なし |
| 王牌 | ドラ表示2枚 + 裏ドラ表示2枚の計4枚のみ → ツモ番は4人打ちで80回、3人打ちで93回 |

## 記事のルールと解釈が分かれた点

**リーチの加点**
記事のルール表は「リーチ = 2点」ですが、記事中の計算例は加算していません。

> 「リーチ・ドラ2」の場合 — ロンアガリ: 6+1+1=8点 / ツモアガリ: (2+1+1)×3=12点

ルール表を正として **リーチ +2点** で実装し、画面右上の「リーチ加点」で
`0点` (記事の計算例と一致) に切り替えられるようにしてあります。
コードからは `DEFAULT_RULES.riichiPoints` で制御します。

## 記事に記載がなく、こちらで決めた挙動

- **嶺上牌**: 山の後ろ (ドラ表示牌の手前) から取り、その分ツモ番が1回減る。
- **リーチ後の暗槓**: ツモ牌で槓が成立し、かつ待ちが変わらない場合のみ許可。
- **一発 / ダブルリーチ**: 記事に記載がないため無し。
- **槍槓**: 未実装 (加槓は他家からロンされない)。フリテンが無いぶん影響は小さい。
- **チョンボ**: UIが不正操作を通さないため発生しない。記事の罰則 (計20点払い) は未実装。
- **親の移動**: 一局精算なので連荘の概念はないが、第一ツモの有利を均すため
  局ごとに親を1つずつ回している。

## 構成

```
src/
  core/            UIから独立した純粋ロジック (テスト対象)
    tiles.ts       牌の表現・136枚の山・ドラ表示牌の巡回
    agari.ts       和了形の判定 (4メンツ1雀頭 / 七対子 / 国士) と待ち
    shanten.ts     シャンテン数と受け入れ (CPU用)
    score.ts       点数計算
    game.ts        1局の進行 (状態機械)
    stakes.ts      仮想レート/レーキの換算 (game.ts はこの層を知らない)
    rng.ts         シード付き乱数 (局の再現用)
  ai/cpu.ts        簡易CPU
  ui/
    tileImage.ts   牌IDと牌画像ファイルの対応
    Tile.tsx       牌1枚。席の向きに応じた回転と外形の縦横入れ替え
    sound.ts       効果音の合成 (Web Audio。外部音源は使わない)
    App.tsx        卓のレイアウトと進行
    RulesPanel.tsx ルールタブ
    TileChart.tsx  開発用の牌見本 (#tiles)
```

シャンテン計算は手牌全体を1回のDFSで探索すると組み合わせが爆発するため、
萬子/筒子/索子/字牌の4グループに分けて分解し、牌姿をキーにメモ化しています。

`core` はDOMに一切依存しないので、オンライン対戦に広げる場合は
そのままサーバ側の権威ロジックとして使えます。

### CPUの方針

役が無いぶん、戦略はほぼ牌効率に集約されます。

- 打牌: シャンテンを落とさない候補のうち受け入れが最大の牌を切る。
  同点なら赤ドラ(5)と対子を残し、孤立字牌から先に切る。
- リーチ: 供託が無く2点と裏ドラが付くため、宣言できるなら常に宣言する。
- ポン/カン: 役が不要なのでシャンテンが進むなら鳴く。
  ただし門前で1シャンテン以内ならリーチの価値を優先して見送る。

## 素材

牌画像は [麻雀豆腐](https://majandofu.com) が無料配布している牌画像素材を使っています
(`public/tiles/`、66×90px)。ファイル名と牌の対応は素直な連番ではないので注意:

| 画像 | 牌 |
|---|---|
| `ji1`〜`ji4` | 東 南 西 北 |
| `ji5` / `ji6` / `ji7` | 發 / 白 / 中 |
| `aka1` / `aka2` / `aka3` | 赤5筒 / 赤5索 / 赤5萬 |

韓麻は5が全て赤ドラなので、5m/5p/5s には常に `aka*` を使っています。

効果音は外部音源を使わず Web Audio API で合成しています (`src/ui/sound.ts`)。
打牌音はノイズ+低音、掛け声はベル音です。

## 仮想レート / レーキ

元ルールはカジノ運用が前提で「レートに応じたコミッション(場代)を審判に支払う」
「局開始時にデポジット額を持っていないと次へ進めない」とされています。
本実装はその**構造だけ**を仮想チップで再現しました (設定タブから変更可)。

| 設定 | 既定値 | 意味 |
|---|---|---|
| レート | 1点 = 100チップ | 点数増減をチップに換算する係数 |
| レーキ | 5% | 和了者の受取から差し引かれ、ハウスへ回る。放銃者の支払いは増えない |
| デポジット | 1,000チップ | 下回った席が出ると次局に進めず、買い直しになる |

`core/stakes.ts` に閉じており、`core/game.ts` はこの層を一切知りません。

### 注意 (賭博性について)

扱うのは**無価値な仮想チップのみ**で、金銭に関わる処理は実装していません。
日本国内で金銭を賭ける形で運用すると賭博罪に該当し得ます。
